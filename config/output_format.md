# 学习笔记输出格式

生成中文学习笔记，保存为 Markdown 格式。

## 模板

```markdown
# {视频标题}

**平台**: YouTube
**时长**: {时长}
**链接**: {链接}

---

## 总结

[2-3 段，用大白话概括视频核心内容]

## Insight

[3-5 个要点，精准、实在、有启发性]

## 概念术语表

[核心概念的定义和解释，术语表性质，方便快速查阅]

## 详细内容

[视频的完整内容，按主题组织，包含论证、案例、细节]

```

## 各部分要求

### 总结
- 2-3 段，用大白话概括视频核心内容
- 让没看过视频的人快速了解"这个视频讲了什么"

### Insight
- 3-5 个要点
- **必须结合用户背景来写**（参考用户背景信息）：
  - 对我有什么启发？
  - 对我的工作/学习有什么帮助？
  - 有什么值得关注的新信息？
- **写作原则**：
  - 具体、可操作、有信息量
  - 一句话说清楚，不绕弯子
  - 不要空泛（"这对行业有深远影响"）
  - 不要务虚（"我们应该拥抱变化"）

### 概念术语表
- 定位：术语表 / 概念速查卡
- 解释原则：
  - 假设读者不懂技术，但很聪明、一点就通
  - 一句话解释清楚
  - 用类比帮助理解，但不要过度简化
- 用表格形式：
  ```
  | 概念 | 解释 |
  |------|------|
  | API | 程序之间的调用接口，像餐厅菜单——你点菜，厨房出菜 |
  | Agent | 能自主决策和行动的 AI，不只是问答，能帮你完成任务 |
  ```

### 详细内容
- 按主题/章节组织视频的全部有价值内容
- 包含论证过程、具体案例、数据支撑
- 保留讲者的独特观点和表达方式
- 这是文档的主体部分，篇幅最长

**核心原则：保留全部有价值的信息，不遗漏任何知识点**

1. 必须读完全部内容再开始写
2. 保留所有知识点、观点、论据、例子、案例、数据
3. 可以精简的：同一观点的多次重复、口水词、广告
4. 按中文逻辑重新组织，不是逐句翻译
5. 专有名词保留英文

长度参考（非硬性要求）：
- 10 分钟视频 → 约 800-1500 字
- 30 分钟视频 → 约 2000-4000 字

## 通用原则

1. **语言**：全部使用中文（专有名词保留英文）
2. **语气**：口语化、易懂，避免学术腔
3. **类比**：用生活化的类比解释技术概念
4. **排版**：善用表格、列表提升可读性
